# Skills — Team Collaboration System

This repository defines a complete team workflow with 6 roles, 5 shared protocols, and an end-to-end delivery process.

## Quick Start

**Step 1** — Read the big picture:
- [`team-workflow.md`](team-workflow.md) — End-to-end process from requirement to delivery (7 phases, communication matrix, phase skip rules, emergency handling)

**Step 2** — Read your role:

| Role | File |
|------|------|
| Product Manager | [`product-manager/product-manager-pro.md`](product-manager/product-manager-pro.md) |
| Tech Lead | [`tech-lead/tech-lead-pro.md`](tech-lead/tech-lead-pro.md) |
| Backend Engineer (Go) | [`software-engineer/golang/golang-senior-pro.md`](software-engineer/golang/golang-senior-pro.md) |
| Frontend Engineer (React) | [`software-engineer/react/react-frontend-engineer-pro.md`](software-engineer/react/react-frontend-engineer-pro.md) |
| QA Engineer | [`qa-engineer/qa-engineer-pro.md`](qa-engineer/qa-engineer-pro.md) |
| DevOps Engineer | [`devops-engineer/devops-engineer-pro.md`](devops-engineer/devops-engineer-pro.md) |

**Step 3** — Reference protocols as needed:

| Protocol | When to Read |
|----------|-------------|
| [`protocols/task-schema.md`](protocols/task-schema.md) | Creating or receiving a task |
| [`protocols/api-contract.md`](protocols/api-contract.md) | Defining or consuming an API |
| [`protocols/test-handoff.md`](protocols/test-handoff.md) | Handing off work to QA |
| [`protocols/bug-report.md`](protocols/bug-report.md) | Reporting or fixing a bug |
| [`protocols/status-sync.md`](protocols/status-sync.md) | Syncing progress or escalating risk |
